From Pixels to Violations: Building a Smart Traffic Monitoring System with Python and YOLO

By Dan 07 September 2025

From Pixels to Violations: Building a Smart Traffic Monitoring System with Python and YOLO

1. Introduction: The Problem and Project Goal

In modern urban environments, ensuring pedestrian safety on crosswalks is a critical challenge. Traffic violations, such as vehicles blocking pedestrian right-of-way, are common and difficult to monitor manually. Inspired by principles from projects like Coursera’s “Fast Prototyping of GenAI Apps,” this project outlines the development of a proof-of-concept (POC) system designed to automatically detect specific traffic violations using computer vision.

The core objective evolved from simple object detection to a more nuanced analysis: Can we automatically identify when a vehicle obstructs a pedestrian’s path on a crosswalk? This required breaking down a complex real-world interaction into smaller, machine-readable steps.

2. Phase 1: Baseline Model for Object Detection

The initial phase focused on establishing a baseline capability: detecting relevant objects in a video feed.

  • Technology Choice: The YOLO (You Only Look Once) model was selected for its high speed and accuracy in real-time object detection.
  • Initial Implementation (vehicle_detection.py): The first script processed video files frame by frame. The primary goal was to successfully identify and classify two main categories: ‘person’ and ‘car’/‘motorcycle’.
  • Feature Enhancement (License Plate Recognition): As an extension, basic Optical Character Recognition (OCR) was successfully implemented to read license plate numbers from detected vehicles. This proved feasible given sufficient video resolution and demonstrated potential for identifying specific violators.

3. Phase 2: Defining the Violation and Region of Interest (ROI)

A significant challenge emerged: simple detection cannot differentiate between a vehicle legally stopped near a crosswalk and one illegally blocking a pedestrian. The system needed to understand context and interaction.

  • Problem Refinement: The key question was “What defines a block?” The proposed solution involves defining a specific high-risk area—the crosswalk—and analyzing object movement within that zone.
  • Defining the ROI: Instead of analyzing the entire image, detection efforts were focused on a Region of Interest (ROI) corresponding to the crosswalk. This significantly reduces computational overhead and filters out irrelevant objects.

4. Phase 3: Real-World Data Acquisition and Challenge Handling

To move from a controlled test video to a more dynamic environment, a data collection strategy was implemented.

  • Data Source (snapshot_grabber.py): A script was developed to capture image snapshots at regular intervals (e.g., every second for 20 minutes) from public government traffic cameras. This provided a realistic dataset for testing.

  • Addressing False Positives: Real-world data introduced new problems. A common issue was the model misidentifying static road markings, such as a large painted arrow, as a ‘person’. While adjusting confidence thresholds is a possible solution, it highlights the limitations of relying purely on classification without context. The ROI approach helps mitigate this by only considering detections in the area where a person is expected to be.

5. Phase 4: Path Analysis for Violation Detection

The final phase focused on implementing the core violation logic by analyzing object movement.

  • Core Concept: A true “block” occurs when a pedestrian’s intended path is obstructed by a vehicle. To model this, we must track the pedestrian’s location over time to establish a direction vector.
  • Implementation (analyze_blocking.py): The logic involves storing the recent coordinates of detected pedestrians. By calculating their direction of travel, the system can project their path. If a vehicle is detected in front of this projected path within the crosswalk ROI, the system flags a potential blocking violation.

6. Conclusion and Future Work

This project successfully demonstrates a proof-of-concept for detecting complex traffic violations by moving beyond simple object detection to contextual analysis within a defined ROI. The process involved iterating from a basic prototype, identifying key challenges like false positives, and developing a logic for path analysis.
